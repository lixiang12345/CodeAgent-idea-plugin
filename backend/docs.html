接口契约

协议版本固定为 1。模型密钥只存在于后端；托管部署通过 OIDC PKCE 签发可撤销的 CodeAgent 会话。

OpenAPI 3.1 JSON Health JSON

端点

GET/health健康、协议版本和默认模型
GET/v1/auth/config发现本地、共享令牌或 OIDC 鉴权模式
GET/v1/auth/authorize启动浏览器 OIDC Authorization Code + PKCE
POST/v1/auth/token交换一次性 code 或轮换 refresh token
GET/v1/me读取当前用户、用量和会话
POST/v1/auth/logout撤销当前会话
GET/v1/models后端允许使用的模型清单
GET/v1/tools后端集成工具和配置状态
POST/v1/tools/{toolName}执行已配置的后端集成工具
GET/v1/configurations/{kind}读取 MCP、命令、Hook、Agent、插件和工具权限配置
PUT/v1/configurations/{kind}/{id}校验并保存账号级产品配置
DELETE/v1/configurations/{kind}/{id}删除产品配置
POST/v1/runs创建 Agent 运行并返回 SSE 流
POST/v1/runs/{runId}/tool-results回传 IDEA 本地工具结果并继续运行
DELETE/v1/runs/{runId}取消运行

请求要点

POST /v1/runs 必须使用 streaming fetch，不能使用只支持 GET 的 EventSource。

SSE 事件：run.started、turn.started、message.delta、assistant.completed、tool.request、tool.completed、run.completed、run.error。

收到 tool.request 后，由 IDEA/JVM 执行本地能力，再向工具结果端点回传。

会话与模型
